Cluster mirroring in Kafka

The Kafka mirroring feature is used for creating the replica of an existing cluster, for example, for the replication of an active datacenter into a passive datacenter. Kafka provides a mirror maker tool for mirroring the source cluster into target cluster.

The following diagram depicts the mirroring tool placement in architectural form:

In this architecture, the job of the mirror tool is to consume the messages from the source cluster and republish them on the target cluster.

A similar approach is used by the Kafka migration tool to migrate from 0.7.x Kafka cluster to 0.8.x Kafka cluster.

Note

For detailed explanation on the mirror maker tool setup and configuration, visit https://cwiki.apache.org/confluence/display/KAFKA/Kafka+mirroring+%28MirrorMaker%29.
